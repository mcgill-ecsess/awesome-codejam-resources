Basics of Text Classification¶

By Charlotte Gils

The goal of this tutorial is to introduce the basic concepts of building a text classification model.

We consider a data set of Medium articles, which can be found at www.kaggle.com.

The goal of the model is to assign a topic tag to an article, based on the article's title, subtitle, author, and publication.

For educational purposes, the original data set has been reduced, and the problem has been simplified:

  • We consider only articles that have a single tag (we will not consider a multilabel classification problem here)
  • Only articles with both a title and a subtitle are considered (note that this impacts the applicability of the model)
  • To further simplify the problem for educational purposes, we retain only the top 4 categories. Note that - as a result - our model is applicable only to articles that are in one of these 4 categories.

A) Tokenize¶

Split the text into its constituent words.

Note that there is no one tokenizer that works for all types of texts; tuning the tokenizer is part of feature engineering.

In addition, the following steps are typically taken:

  • Convert all words to lower case, although we may want to retain all capitalized words
  • Remove stop words
  • Stemming (Optional: Sometimes this is useful but not always; there are multiple stemmers)
  • Include higher-than 1 ngrams
  • Remove words with less than 3 characters
In [3]:
import nltk
from nltk.stem.snowball import SnowballStemmer
from nltk.corpus import stopwords
from nltk import ngrams
sw = set(stopwords.words('english'))
stemmer = SnowballStemmer("english")
pattern =  "[\'A-z\-]+|[@\#]\w+"

def tokenize(text, stem=False, n_grams=1 ):
    if text is None:
        return []
    tokens = [t.lower() for t in nltk.regexp_tokenize(text, pattern)]
    
    #remove stopwords and words with less than 3 charactesr
    tokens = [t for t in tokens if t not in sw and len(t) > 2]
    
    if stem == True:
        tokens = [stemmer.stem(t) for t in tokens]
    
    if n_grams > 1:
        copy_tokens = tokens.copy()
        for n in range(2, n_grams+1):
            tokens.extend(["_".join(list(g)) for g in ngrams(copy_tokens, n)])
  
    return tokens

C) Tfidf¶

Tfidf stands for "term frequency-inverse document frequency":

  • Term frequency (Tf): Give more weight to words that appear more frequently within a document
  • Inverse document frequency (idf): Reduce the weight of words that appear frequently across all documents

Definitions:

  • Tf(word): Number of times "word" appears in the document
  • idf(word): Log( Number of documents / Number of documents where "word" appears))
In [4]:
from sklearn import feature_extraction

def generate_features(args, df_train, df_test, min_word_ct = 1,):
    tfidfVect = feature_extraction.text.TfidfVectorizer(
                tokenizer=lambda text: tokenize(text, **args), 
                lowercase=False,  min_df = min_word_ct)

    X_train =  tfidfVect.fit_transform(df_train["all_text"])
    X_test = tfidfVect.transform(df_test["all_text"])
     
    #scale to mean 0 and unit variance (allows to compare magnitudes of coefficients)
    scaler = preprocessing.StandardScaler(with_mean=False)
    X_train = sparse.csr_matrix(scaler.fit_transform(X_train.toarray()))
    X_test = sparse.csr_matrix(scaler.transform(X_test.toarray()))
    
    vocab = tfidfVect.vocabulary_
    
    return X_train, X_test, vocab

Note that we have combined all text fields into one field, but note that it might be advantageous to retain the information on the field (title, subtitle, etc) where a token appeared (we will not do that here).

III Evaluation Metrics¶

Before we move to training a model, we need to decide how to evaluate the model!

In multi-class classification, one typically assesses performance using

Precision(c): # samples correctly predicted to be in class c/ # samples predicted to be in class c

Recall(c): # samples correctly predicted to be in class c / # samples actually in class c

The weighted F1-score is the sum of

2 precision(c) recall(c)/ (precision(c)+ recall(c)),

weighted by the sum of samples in class c

We also assess the confusion matrix which is a table of counts of actual classes (columns) vs predicted classes (rows) to identify which classes were "mis-classified" into what other classes.

Note that business requirements typically dictate

  • the appropriate evaluation metrics
  • how high the evaluation must be

A model may be tuned to optimize the chosen metric (typically at the expense of other metrics; e.g., recall vs precision, class X vs class Y).

Below, we will tune our model based on weighted F1-score: I.e., we give equal weight to optimizing precision and recall, and give weight to classes based on their respective proportions in the data.

IV Train Classification Model¶

from sklearn import linear_model
from sklearn import metrics
from sklearn import preprocessing
from sklearn import svm
from sklearn.externals import joblib
import matplotlib.pyplot as plt
%matplotlib inline
import seaborn as sns
from scipy import sparse

def train_model(df_train, df_test, args, min_word_ct, alg = "log_reg",
                param_grid = { "C": np.logspace(-3, 0, 4), "penalty" : ["l1"]}, 
                print_conf = False):

    #generate features
    X_train, X_test, vocab = generate_features(args, df_train, df_test, min_word_ct)
    inv_vocab = {v: k for k, v in vocab.items()}
    print("Number of features: {}".format( X_train.shape[1]))
    
    #specify algorithm 
    if alg == "log_reg":
        estimator = linear_model.LogisticRegression(solver = "saga", class_weight="balanced", multi_class = "ovr", max_iter=500)
    elif alg == "svm":
        estimator = svm.LinearSVC(class_weight="balanced", dual=False, multi_class = "ovr", max_iter=500)
 
    #Determine best hyperparameter using 5-fold cross-validation
    cv = model_selection.StratifiedKFold(n_splits=5, shuffle=True)
    model = model_selection.GridSearchCV(estimator, param_grid, cv=cv, scoring="f1_weighted", n_jobs=-1)
    model.fit(X_train, df_train["label"])
    print ("Best mean CV weighted f1-score is {} at parameters {}".format(round(model.best_score_, 3), model.best_params_))
    
    #For each class, print largest coefficients
    coeff = model.best_estimator_.coef_
    print ("Top coefficients")
    df_coeff = pd.DataFrame()
    for j, l in enumerate(np.unique(df_train["label"])):
        coeff_l = coeff[j]
        sorted_indices = np.argsort(np.abs(coeff_l))
        nonzero_c = np.sum([0 if c==0 else 1 for c in coeff_l])
        df_coeff[l] = [inv_vocab[i] + " " + str( round(coeff_l[i], 2)) for i in sorted_indices[::-1][:10]]
    print (df_coeff)
    print ("\nNon-zero coefficients: {} out of {}".format(nonzero_c, coeff.shape[0]*coeff.shape[1]))
    
    #Evaluate the model on the test set
    print ("\nEvaluation scores on test set: ")
    preds = model.predict(X_test)
    print (metrics.classification_report(df_test["label"], preds, target_names = np.unique(df_test["label"])))
    
    if print_conf == True:
        conf = metrics.confusion_matrix(df_test["label"], preds)
        df_conf = pd.DataFrame(conf, index = np.unique(df_test["label"]), columns =  np.unique(df_test["label"]))
        plt.figure(figsize = (10,7))
        sns.heatmap(df_conf.astype(int), annot=True)
    
    return model, preds

Regularization (strength: parameter C) imposes restrictions on the complexity of the model.

For l2-regularization, features that are not very relevant have smaller impact (smaller coefficients). For l1-regularization, "irrelevant" features will have coefficients with value 0. See below: Only a small proportion of coefficients are non-zero.

B) Support Vector Machine¶

Let's try a linear support vector machine (SVM) algorithm. As the decision boundaries learned by logistic regression and support vector machine are both linear, we typically would not expect huge differences in evaluation scores. As opposed to SVM, logistic regression naturally provides a probabilistic output (probabilities that a documents is in each class), which can be useful in practice.

The evaluation scores are slightly inferior to those of the logistic regression.

C) Recurrent Neural Network¶

The features used in the above models do not consider the order of words within the document (1-grams), or only to the extent of neighbouring words (2-grams). (Including higher-order n-grams quickly explodes the number of features, which may decrease evaluation scores.)

A recurrent neural network takes into account the order of tokens in the texts.

We adapt https://www.kdnuggets.com/2018/07/overview-benchmark-deep-learning-models-text-classification.html to our dataset. See also https://github.com/keras-team/keras/tree/master/examples.

Some comments on the model:

  • The embedding layer: A vector based representation of the words (as opposed to the bag-of-words representation). Similar words have smaller euclidean distance between their respective vectors. Here, we use a random initialization for the word embeddings, which are then iteratively improved during training.
  • Dropout: Randomly drop (set to 0) 30% of embeddings at each update to reduce overfitting: This takes the role of regularization.
  • Gated recurrent unit (GRU): The actual RNN architecture. GRU's are an (faster to train) alternative to LSTM's.
  • Pooling: The recurrent network outputs latent vectors for each word (that contain long-range information as a result of the RNN). To extract the most important information in the latent vectors, a concatenation of their average and their maximum (elementwise) are considered.
  • Categorical cross entropy is in fact the same loss function as in a multinomial logistic regression
  • Metrics: For reasons of simplicity we tune using accuracy (as our classes are relatively balanced, this should yield the same results as tuning with weighted f1-score).
  • Adam optimizer (adaptive learning rates), empirically shown to lead to faster convergence than other stochastic optimization methods
  • Softmax output: This is the same transformation to a probabilistic output as in the case of multinomial logistic regression.
In [22]:
def get_rnn_model(embedding_dim= 300, embedding_matrix=np.random.random((MAX_NB_WORDS, 300))):
    
    inp = Input(shape=(MAX_LENGTH, ))
    x = Embedding(input_dim=MAX_NB_WORDS, output_dim=embedding_dim, input_length=MAX_LENGTH, 
                  weights=[embedding_matrix], trainable=True)(inp)
    x = SpatialDropout1D(0.3)(x)   
    x = Bidirectional(GRU(100, return_sequences=True))(x)
    avg_pool = GlobalAveragePooling1D()(x)
    max_pool = GlobalMaxPooling1D()(x)
    conc = concatenate([avg_pool, max_pool])
    outp = Dense(6, activation="softmax")(conc)
    
    model = Model(inputs=inp, outputs=outp)
    model.compile(loss='categorical_crossentropy',
                  optimizer='adam',
                  metrics=["accuracy"]) 
    return model

rnn_simple_model = get_rnn_model()

D) Recurrent neural network with pretrained glove-embeddings¶

Instead of using randomly initialized word embeddings, we used word embeddings that were pre-trained on a large corpus (wikipedia): Thus, we use external knowledge about semantical and syntactical relationships between words.

def get_coefs(word, *arr):
    try:
        return word, np.asarray(arr, dtype='float32')
    except:
        return None, None

    v = embeddings_index[k]
    try:
        if v.shape != (embed_size, ):
            embeddings_index.pop(k)
    except:
        pass

"""
initialize embedding matrix to normal dist with mean and std of all embeddings (some tokens do not have embedding vector)
"""
values = list(embeddings_index.values())
all_embs = np.stack(values)

emb_mean, emb_std = all_embs.mean(), all_embs.std()
word_index = tokenizer.word_index
nb_words = MAX_NB_WORDS
embedding_matrix = np.random.normal(emb_mean, emb_std, (nb_words, embed_size))

oov = 0
for word, i in word_index.items():
    if i >= MAX_NB_WORDS: continue
    embedding_vector = embeddings_index.get(word)
    if embedding_vector is not None:
        embedding_matrix[i] = embedding_vector
    else:
        oov += 1

print("Number of words {}, {} of which not have pre-trained embedding vector".format(len(word_index), oov))
